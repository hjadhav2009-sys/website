9. Brand Rules for Developers
These rules must be followed on every page of the website. No exceptions without approval.
Must Do — Green Rules
✓
Rule — must always follow
✓
All headings use Plus Jakarta Sans. All body text uses DM Sans. Never mix or substitute.
✓
All primary action buttons must be #1B4F9E blue background with white text.
✓
All product cards: white background, 1px solid #E2E8F0 border, 16px border radius.
✓
Cards on hover: translateY(-4px) transform + var(--shadow-md) blue shadow.
✓
All prices displayed in #1B4F9E blue, Plus Jakarta Sans, 700 weight.
✓
Sections alternate backgrounds: white → off-white (#F8FAFF) → white → etc.
✓
Footer background is always #0A0A1A (rich black) with white/gray text.
✓
Max content width is 1200px, always centred with margin: 0 auto.
✓
All images in cards use object-fit: cover to maintain consistent proportions.
✓
All transitions use 0.2s ease for smooth professional feel.
✓
Product images on cards: fixed height (220px desktop, 180px mobile), object-fit cover.
✓
Sale badges: #E53E3E red, white text, 4px border radius, 11px font size.
✓
All form inputs: 1.5px border #E2E8F0, 10px border radius, focus border #1B4F9E.
✓
Announcement bar: always #1B4F9E blue background, white text, 38px height.
Must Not Do — Red Rules
✗
Never do this
✗
Never use more than 3 colours in a single section (blue, white, dark only).
✗
Never use any font other than Plus Jakarta Sans and DM Sans.
✗
Never add rainbow or multi-colour gradients. Only blue gradients allowed.
✗
Never use red (#E53E3E) for anything except sale badges and flash deals.
✗
Never use amber/yellow (#F59E0B) for anything except star ratings.
✗
Never use CSS to override Elementor homepage sections — they control different areas.
✗
Never put any Additional CSS that styles homepage elements — Elementor handles homepage.
✗
Never use black buttons. Primary buttons are always blue.
✗
Never use fonts from anywhere other than Google Fonts.
✗
Never use decorative shadows (only functional shadows as listed in the shadow system).
