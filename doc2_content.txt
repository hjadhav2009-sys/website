8.3 Passive GPS and Location System — Complete Technical Specification
🔴  This is the most technically critical feature. The finder must NEVER see any location permission popup. Location must be captured silently and automatically the moment the QR page loads.

LAYER 1 — IP Geolocation (Always Works — Zero Permission — Instant)
This runs on every single QR page load. No exceptions.
When finder opens QR page — PHP on our server captures their IP address from $_SERVER['REMOTE_ADDR']
Server makes HTTP call to ip-api.com (FREE — 45 requests per minute — no API key needed)
ip-api.com returns: city, region, country, latitude, longitude, ISP, timezone
Accuracy: 500 metres to 2 kilometres — enough to identify the neighbourhood
All data saved instantly to scan_events database table
If owner is on Standard or Corporate plan — WhatsApp message sent immediately to owner

LAYER 2 — Browser GPS (Silent Attempt — Gets Precise Location if Browser Allows)
AFTER the page finishes loading visually — JavaScript runs silently in background
Code: navigator.geolocation.getCurrentPosition(successCallback, errorCallback, {timeout: 5000})
Timeout: 5 seconds — if browser does not respond — Layer 1 data is used
If browser allows automatically (Chrome Android often does) — gets GPS within 10-50 metres
If browser asks permission popup and finder denies — no problem — Layer 1 already captured
If GPS succeeds — updates the scan_events record with precise coordinates
Owner gets updated WhatsApp message: Precise location updated — tap to view

LAYER 3 — Device and Network Fingerprint (Extra Confidence)
JavaScript captures silently: device type, OS version, browser name, screen resolution
JavaScript captures: timezone from Intl.DateTimeFormat().resolvedOptions().timeZone
Timezone confirms city — example: Asia/Kolkata confirms India
ISP name from ip-api confirms the area — example: Jio Mumbai confirms Mumbai
All stored in scan_events table as device_fingerprint JSON column

ONE HOUR LOCATION WINDOW — Privacy and No Misuse
Exact GPS coordinates (latitude and longitude) are visible to owner for exactly 1 hour from scan time
After 1 hour — coordinates are hidden from dashboard — replaced with city name only
After 24 hours — PHP cron job permanently deletes coordinates from database
City name and timestamp are stored permanently in scan history — coordinates are not
THEMENGIFT admin can also see location for same 1 hour window only — no misuse possible
Owner dashboard shows: LIVE — 12 minutes ago — Bandra West Mumbai — View on Map (clickable for 48 mins more)
Expired: Scanned 3 days ago — Andheri Mumbai — Location Expired

MAP TECHNOLOGY — OpenStreetMap with Leaflet.js (100% Free Forever)
Google Maps embed API costs money and has billing limits — DO NOT USE for map display
Use OpenStreetMap tiles with Leaflet.js JavaScript library — completely FREE, no API key needed
Leaflet.js CDN: https://unpkg.com/leaflet@1.9.4/dist/leaflet.js
OpenStreetMap tiles: https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png
WhatsApp notification to owner uses Google Maps LINK format — this is FREE: maps.google.com/?q={lat},{lng}
Owner taps WhatsApp link — opens Google Maps app on their phone — shows pin location
Owner dashboard map — OpenStreetMap with Leaflet showing scan pin — beautiful and free
✅  WhatsApp LINK to Google Maps is free. Only EMBEDDING Google Maps on a webpage costs money. We use Leaflet for webpage display and Google Maps link in WhatsApp. Best of both worlds at zero cost.
